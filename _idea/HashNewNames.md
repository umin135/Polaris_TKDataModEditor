Each Move's name is hashed to generate the "name_key" field that we see in "tk_move". Originally, each move used to have a "name" in Tekken 7 (and prior) but Bandai Namco removed it and replaced it with the "name_key" field in Tekken 8.
Now we have that hash function pulled out from the game's disassembled code, Bamco calls it "Kamui Hash" so we'll also call it that.
Now, the current editor keeps track of newly added moves, what needs to be done is that whenever a moveset is saved and exported, the new name should be hashed using our Kamui hasher and stored as the "name_key." This helps achieve the rule that each move's name key is unique throughout. The same principle applied to move names in Tekken 7 and prior games.

Not only that, this hash function can be used to generate the ordinal id field (offset 0xF0) of a move. Said value is generated by merging the character's 3-letter code with the name.
E.g, The Heat Dash move `Co_ZoneDash_Cancel` for Kazuya would be combined to get `GRL_Co_ZoneDash_Cancel`, which will generate Kamui hash `0xa0ee16ea`.
